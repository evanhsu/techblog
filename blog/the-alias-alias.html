The Alias Alias

May 16, 2022 · 2 min read
Evan Hsu
Author

Store terminal commands as aliases​

You can create "aliases" for common terminal commands (or commands that are hard to remember).

They look like this:

alias gl='git log --oneline --decorate --all --graph'
TL;DR

Make sure to include "the alias alias":

alias aliases="vim ~/.aliases && source ~/.zshrc"

Where to put your aliases​

Aliases like the one above need to be "sourced" in your terminal settings. The correct file will differ depending on which terminal you use:

TerminalFile
Bash~/.profile OR ~/.bashrc OR ~/.bash_profile
zshell~/.zshrc
OtherWhy are you here?

I highly recommend creating a new file that just stores aliases: ~/.aliases and then loading ("sourcing") this file from within your profile or "rc" file:

# ~/.zshrc or other 'rc' file
if [ -f ~/.aliases ]; then
    . ~/.aliases
fi

Then your ~/.aliases file will just look like this:

# ~/.aliases

# Aliases for Git
alias gs='git status'
alias gl='git log --oneline --decorate --all --graph'
alias gll='git log --oneline --all --graph --pretty=format:"%C(yellow)%h %C(auto)%d %C(reset)- %s {%C(green)%an, %ar%C(reset)}"'
alias gco='git checkout'
alias gpr='git pull --rebase'
alias grh='git reset --hard'

# Aliases for Tmux
alias tml='tmux list-sessions'
alias tma='tmux attach -t '
alias tmn='tmux new -s '

# VS Code
alias c='code .'

# Docker
alias dc="docker-compose"

# Kubernetes
alias kc="kubectl"
alias kccon="kubectl config current-context"
alias kcuse="kubectl config use-context"

The best alias​

alias aliases="vim ~/.aliases && source ~/.zshrc"

Make sure you include this one. This makes it super easy to add a new alias or check to see what aliases you already have. At your command line, just type aliases and you're in.
